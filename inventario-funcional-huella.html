📌 Resumen Ejecutivo

Huella es la plataforma de gestión de cobranza y procesos operativos de arrendamiento de Seguros Bolívar. Opera sobre Oracle B2C Service (Oracle Service Cloud), provista por ImagineCX y soportada por Stefanini.

Gestiona el ciclo completo de cobranza jurídica y pre-jurídica, desistimientos, desocupaciones, liquidaciones y devoluciones para contratos de arrendamiento, con 1,640 usuarios activos distribuidos en 40 grupos organizacionales.

🔄 Procesos Principales

⚖️

Cobranza Jurídica

Gestión de cobros mediante procesos legales. Incluye radicación, seguimiento de demandas, sentencias y ejecución de fallos judiciales.

📋

Cobranza Pre-Jurídica

Gestión preventiva antes de acción legal. Contacto con deudores, acuerdos de pago, negociación de condiciones y seguimiento de compromisos.

🚪

Desistimiento

Proceso de terminación voluntaria de contratos. Gestión de avisos, validación de condiciones y cierre administrativo del contrato.

🏠

Desocupaciones

Gestión de entrega de inmuebles arrendados. Coordinación de inspecciones, actas de entrega y liquidación final del contrato.

🎯 Impacto de la Migración

¿Qué se gana con la migración?

  • Eliminación de dependencia de Oracle DB (stack legacy) → PostgreSQL (estándar aprobado)
  • Reducción de costos de licenciamiento Oracle Service Cloud
  • Mayor autonomía tecnológica (sin dependencia de ImagineCX/Stefanini)
  • Integración nativa con el ecosistema cloud AWS de la organización
  • Modernización de la experiencia de usuario (UX actual limitada por Oracle Agent Desktop)
  • Capacidad de escalar y personalizar sin restricciones de plataforma
  • Observabilidad completa con Datadog (APM, logs, métricas)
  • CI/CD con GitHub Actions (actualmente sin pipeline automatizado)

⚡ SAI — Sistema de Administración de Inmuebles

Crítica SAI es la fuente de verdad de toda la información de cobranza. Alimenta a Huella con siniestros, pólizas, contratos, amparos, aumentos y estados de pago.

Flujos de datos

FlujoDirecciónDatosFrecuencia
Creación de siniestrosSAI → HuellaSiniestro + póliza + amparos + aumentosTiempo real
Actualización de estadosSAI → HuellaEstado siniestro, estado pago, novedadesTiempo real
Creación de tercerosSAI → HuellaTipo documento + número documentoCon cada siniestro
Bitácora de gestiónHuella → SAIRegistros de gestión de agentesTiempo real
Actualización de estadosHuella → SAICambios de estado de siniestrosTiempo real

Punto de fallo: El bus de servicios es el primer punto de diagnóstico cuando hay desincronización. Contiene la mayor parte de la lógica de integración.

👥 Base de Terceros

Crítica Base centralizada del grupo empresarial con información de personas naturales y jurídicas (nombres, contacto, ubicación).

Comportamiento especial

  • No sobreescribe datos originales: Actualizaciones de Huella se guardan en espacio dedicado
  • Datos compartidos: Otras empresas del grupo acceden al repositorio genérico
  • Dato faltante = "N": Si un tercero muestra "N" en nombre, Huella aún no consultó la base
  • Trigger: Huella consulta cuando SAI crea un siniestro con terceros nuevos

⚖️ Módulo Jurídico

Alta Sistema on-premise que envía información de procesos judiciales a Huella. Unidireccional — Huella no modifica ni devuelve datos.

Carga contratos y datos de procesos judiciales activos. Toda comunicación pasa por el bus de servicios.

🚀 Microservicio ImagineCX

Alta Microservicio externo (~2 años en producción) que orquesta paz y salvo, terminación de siniestros y desocupación.

Características

  • Corre fuera de Huella — no depende de su carga
  • Comunicación casi inmediata (resuelve latencia anterior Huella↔SAI)
  • Mantenido por ImagineCX

Procesos que orquesta

1. Paz y Salvo  |  2. Terminación de siniestros  |  3. Desocupación

⚠️ Consideraciones para la Migración

  • Volumetría alta: 789,717 siniestros requieren estrategia de migración por lotes con validación de integridad.
  • Modelo de datos: Siniestro vinculado a Solicitud, Arrendatario, Tipo Póliza y Amparo — relaciones que deben preservarse.
  • Flujo PYS: Existe un flujo de envío programado de Paz y Salvo con bloqueo que debe replicarse.
  • 23 reportes especializados: Múltiples vistas de negocio que deben cubrirse en la nueva plataforma.
  • Búsqueda por reporte: En la nueva plataforma, implementar como vista de búsqueda dedicada con filtros dinámicos (mejor UX).
  • Paginación masiva: 10,000 registros por página — considerar paginación server-side con cursor en la nueva implementación.

Sincronización dual, validación de paridad, cutover con rollback plan. Descomisionar Oracle Service Cloud.

⚡ Resumen de Business Rules

Las Business Rules de Oracle Service Cloud son reglas automatizadas que se ejecutan cuando ocurren eventos en los objetos del sistema (crear, editar, cambiar estado). Se extrajeron exitosamente usando el endpoint API interno del AgentWeb.

Hallazgo principal: Solo el objeto Bitácora (Incident) tiene reglas configuradas. Los 9 objetos restantes están vacíos.

🔧 Endpoint API Descubierto (para automatización)

Se descubrió el endpoint interno del AgentWeb que permite exportar Business Rules programáticamente:

POST /AgentWeb/api/contextmanager/v.{version}/businessRulesContext/exportRuleBase
Body: {
  "rulebaseId": {classId},
  "classId": {classId},
  "entityType": "BusinessRuleRulebaseLive",
  "exportFormat": "delimited",
  "ruleExport": true
}

Polling: GET .../businessRulesContext/{classId}/export/BusinessRuleRulebaseLive/status
Descarga: POST .../businessRulesContext/{classId}/export/BusinessRuleRulebaseLive/delimited

Nota: Requiere sesión autenticada en AgentWeb. El rulebaseId coincide con classId excepto para Bitácora (rulebaseId=30317582).
